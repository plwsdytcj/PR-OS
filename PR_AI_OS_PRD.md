Demo 名称：**AI 媒介经理 Copilot**  
文档版本：v0.1  
日期：2026-06-03

---

## 1. 产品一句话

**PR AI OS 是 PR 行业的 AI 操作系统层，帮助品牌输入产品、预算和传播目标后，自动理解传播需求、评估内容节点、匹配博主/KOL、生成投放策略、推演舆论风险，并把传播任务分发给最合适的内容创作者。**

---

## 2. 产品定位

### 2.1 我们不是做什么

PR AI OS 不是：

- 另一个 KOL 资源库；
- 另一个媒介报价表；
- 另一个软广交易平台；
- 另一个达人入驻平台；
- 另一个巨量星图、蒲公英、微播易。

这些产品主要解决“找人、下单、交易”的问题。

### 2.2 我们真正做什么

PR AI OS 要解决的是 PR 行业更底层的判断问题：

- 这个产品应该怎么传播？
- 哪些内容节点适合这个产品？
- 哪些博主真正有商业价值？
- 哪些博主只是数据好看？
- 这笔预算应该怎么分？
- 这个报价合理吗？
- 软广、预热、试用、硬广应该怎么衔接？
- 评论区和舆论风险在哪里？
- 怎么把投后案例沉淀成下一次可复用的资产？

因此，PR AI OS 的核心不是“达人库”，而是：

> **品牌传播需求的理解层 + 内容节点的价值评估层 + 传播任务的智能分发层 + 舆论风险的推演层。**

## 3. 背景与机会

### 3.1 PR 行业的真实工作方式

当前 PR 和 KOL 投放大量依赖：

- 媒介个人经验；
- Excel 刊例表；
- 微信群和私域资源；
- 过往合作记忆；
- 人工判断报价；
- 人工筛选博主；
- 人工写推荐理由；
- 投后零散复盘。

这些能力很有价值，但存在明显问题：

- 经验无法规模化；
- 新人无法快速复制老媒介判断；
- 博主商业价值难以被结构化表达；
- 甲方不知道报价是否合理；
- 投放前缺少推演；
- 投后案例没有回流成系统资产。

### 3.2 核心机会

AI 可以把 PR 行业原本分散在表格、聊天记录、案例、报价、主观经验里的信息，转化成可计算、可匹配、可推演、可复盘的系统资产。

这意味着平台可以同时服务两侧：

- **品牌方**：获得更好的传播决策；
- **博主方**：沉淀和提升自己的商业价值；
- **平台方**：形成跨行业、跨平台、跨案例的 PR 决策资产。

## 4. 核心用户

### 4.1 品牌方 / 甲方市场部

典型用户：

- 品牌市场负责人；
- PR 负责人；
- 新品上市负责人；

- 我这个产品应该找什么类型的博主？

- 视频号创作者；
- 垂直领域内容作者；

他们关心：

- 我的商业价值怎么被品牌理解？
- 我过往合作案例如何变成背书？
- 我适合哪些行业合作？
- 我该怎么报价？
- 我如何进入领域榜单？
- 我如何获得更匹配的品牌机会？

### 4.3 PR 公司 / 媒介团队

典型用户：

- PR 公司媒介经理；
- 内容营销代理商；
- MCN 商务团队；
- 品牌外包媒介团队。

他们关心：

- 如何更快筛人？
- 如何更快生成 KOL 推荐表？
- 如何让新人复制老媒介经验？
- 如何降低无效沟通？
- 如何把案例和资源沉淀成团队资产？

## 5. 产品形态

PR AI OS 是一个双侧平台，核心由三层组成。

### 5.1 博主商业资产层

博主上传：

- 账号信息；
- 平台链接；
- 内容领域；
- 粉丝量级；
- 报价区间；
- 历史品牌合作案例；
- 内容链接；
- 效果数据；
- 可接合作类型；
- 档期和合作偏好。

系统输出：

- 博主商业画像；
- 行业适配标签；
- 内容能力标签；
- 传播阶段标签；
- 预算适配标签；
- 风险标签；
- 领域榜单排名；
- 对甲方可见的商业主页。

### 5.2 品牌传播决策层

甲方输入：

- 产品信息；
- 预算；
- 目标人群；
- 传播阶段；
- 传播目标；
- 平台偏好；
- 内容调性；
- 是否需要试用、预热、预售或硬广转化。

系统输出：

- brief 解析；
- 传播策略；
- 平台建议；
- 博主组合推荐；
- 匹配评分；
- 推荐理由；
- 风险提示；
- 报价判断；
- 预算分配；
- 投放节奏；
- 可导出的方案。

### 5.3 智能匹配与推演层

平台在中间完成：

- 品牌 brief 理解；
- 博主商业价值评估；
- 领域榜单生成；
- KOL/KOC 组合推荐；
- 软广到硬广路径设计；
- 投放前舆论压力测试；
- 合作需求分发；
- 投后案例回流。

## 6. 双侧 User Story

### 6.1 博主侧

#### Story B1：博主建立商业档案

作为一个博主，我希望上传自己的账号信息、平台链接、粉丝量、内容领域和报价区间，这样平台可以帮我建立一个可被品牌理解的商业档案。

验收标准：

- 博主可以填写基础信息；
- 博主可以添加多个平台账号；
- 系统可以生成基础商业主页；
- 系统可以保存报价、领域、合作偏好。

#### Story B2：博主上传品牌合作案例

作为一个博主，我希望上传过往品牌合作案例，包括品牌名称、行业、合作形式、内容链接、效果数据和截图说明，这样平台可以证明我的商业能力。

验收标准：

- 博主可以新增合作案例；
- 每个案例可关联行业、品牌、平台、内容形式；
- 支持填写曝光、互动、转化、评论反馈等数据；
- 系统可以从案例中提取商业标签。

#### Story B3：博主获得 AI 商业标签

作为一个博主，我希望平台根据我的内容和案例，自动生成商业标签，这样品牌可以知道我适合什么合作。

验收标准：

- 系统生成行业适配标签；
- 系统生成内容能力标签；
- 系统生成适合传播阶段；
- 系统生成适合预算层级；
- 系统生成风险提示；
- 博主可以申请人工修正或补充。

#### Story B4：博主进入领域榜单

作为一个博主，我希望进入相关领域榜单，这样可以提高我在品牌方心中的商业价值。

验收标准：

- 系统按行业/场景生成榜单；
- 榜单不只按粉丝数排序；
- 榜单考虑案例、内容能力、报价、风险和行业匹配；
- 博主可以看到自己进入了哪些榜单。

#### Story B5：博主接收匹配的品牌需求

作为一个博主，我希望收到与我领域、内容能力和报价匹配的品牌需求，而不是无关合作。

验收标准：

- 系统可以向博主推送品牌 brief；
- 博主可以查看合作目标、预算区间和内容要求；
- 博主可以选择感兴趣、不感兴趣或需要沟通；
- 平台记录博主响应状态。

### 6.2 甲方侧

#### Story A1：甲方输入传播 brief

作为甲方市场负责人，我希望输入产品、预算、目标人群、传播阶段和投放目标，这样系统可以理解我的传播需求。

验收标准：

- 支持自然语言输入；
- 支持结构化填写预算、平台、目标、阶段；
- 系统能解析出行业、产品、人群、预算和内容方向；
- 支持手动修改解析结果。

#### Story A2：甲方获得传播策略

作为甲方，我希望系统先告诉我应该怎么投，而不是直接给我一堆达人名单。

验收标准：

- 系统输出传播目标拆解；
- 系统输出平台策略；
- 系统输出软广、预热、硬广的节奏建议；
- 系统指出不建议投入的方向；
- 系统给出预算大类分配。

#### Story A3：甲方查看推荐博主组合

作为甲方，我希望看到系统推荐的博主组合，并理解每个博主为什么适合。

验收标准：

- 每个博主有匹配分；
- 每个博主有推荐等级；
- 每个博主有推荐角色；
- 每个博主有历史案例；
- 每个博主有建议内容形式；
- 每个博主有风险提示；
- 每个博主有报价判断。

#### Story A4：甲方查看领域榜单

作为甲方，我希望查看某个传播场景下的领域榜单，这样我可以快速了解这个领域谁更值得关注。

验收标准：

- 支持按行业查看榜单；
- 支持按传播目标查看榜单；
- 支持按平台查看榜单；
- 支持查看博主进入榜单的原因；
- 支持收藏或加入候选名单。

#### Story A5：甲方付费解锁深度信息

作为甲方，我希望先看到部分推荐结果，再付费解锁完整推荐理由、报价建议、联系方式或需求推送能力。

验收标准：

- 未付费可查看部分榜单或模糊信息；
- 付费后可查看完整推荐列表；
- 付费后可查看完整匹配理由；
- 付费后可查看案例详情、报价区间和风险分析；
- 付费后可向博主推送合作 brief。

#### Story A6：甲方一键推送合作需求

作为甲方，我希望把合作需求推送给匹配的博主，让博主主动响应。

验收标准：

- 甲方可以选择推荐列表中的博主；
- 系统生成标准化合作 brief；
- 系统推送给博主；
- 博主可以响应；
- 甲方可以查看响应状态。

#### Story A7：甲方导出投放方案

作为甲方，我希望系统生成一份可以给老板、团队或代理商看的投放方案。

验收标准：

- 输出 Markdown 方案；
- 包含传播目标、平台策略、达人组合、预算分配；
- 包含内容方向、投放节奏、风险提示；
- 支持导出 Excel 推荐表；
- 后续支持导出 PDF/PPT。

```text
        ↓

        ↓

        ↓
平台辅助生成投放方案
        ↓
投后案例回流
        ↓
博主商业价值和平台数据资产增强
```

### 8.2 品牌 Brief 输入器

功能：

- 预算输入；
- 目标人群；
- 传播阶段；
- 平台偏好；
- 投放目标；
- 是否需要试用/预热/预售/硬广；
- AI 解析结果确认。

### 8.3 AI 博主商业价值引擎

功能：

- 从账号信息和案例中提取标签；
- 判断博主适合的行业；
- 判断内容能力；
- 判断商业角色；
- 判断预算适配；
- 识别风险；
- 生成商业总结。

### 8.4 领域榜单

功能：

- 按行业生成榜单；
- 按传播目标生成榜单；
- 按平台生成榜单；
- 按预算层级生成榜单；
- 支持榜单解释；
- 支持榜单更新。

榜单示例：

- 新能源汽车新品预热达人榜；
- AI 软件品牌破圈达人榜；
- 高端护肤小红书种草达人榜；
- 电影提前观影口碑传播榜；
- 职场效率工具推荐达人榜。

### 8.5 KOL 匹配评分

评分维度：

| 维度 | 权重 | 说明 |
|---|---:|---|
| 行业匹配度 | 20% | 博主内容领域是否适合该行业 |
| 人群匹配度 | 15% | 博主受众是否接近目标用户 |
| 内容能力匹配度 | 15% | 博主是否能生产所需内容 |
| 传播阶段匹配度 | 15% | 是否适合预热、种草、背书、转化 |
| 案例背书 | 10% | 是否有同类品牌或相似案例 |
| 报价合理性 | 10% | 报价是否与粉丝量、案例和内容能力匹配 |
| 风险控制 | 10% | 是否存在舆情、广告密度、调性不稳等风险 |
| 响应概率 | 5% | 博主是否可能响应该类需求 |

推荐等级：

| 分数 | 等级 |
|---:|---|
| 85-100 | 强推荐 |
| 70-84 | 推荐 |
| 55-69 | 备选 |
| 40-54 | 谨慎使用 |
| 0-39 | 不推荐 |

### 8.6 预算分配

功能：

- 按平台分配；
- 按博主层级分配；
- 按传播阶段分配；
- 按内容形式分配；
- 保留机动预算；
- 输出不建议投入方向。

### 8.7 投放推演

功能：

- 对不同方案做传播压力测试；
- 模拟目标用户反馈；
- 模拟评论区风险；
- 模拟竞品观察者反应；
- 输出正向反馈、负向反馈、风险点和优化建议。

第一版可以用 LLM 多角色模拟，不承诺真实预测 ROI。

## 12. Demo 场景

### 12.1 新能源汽车新品上市

品牌输入：

```text
我们是新能源汽车品牌，预算 50 万，准备做新能源 SUV 新品上市预热。
目标用户是 25-40 岁一二线城市男性，希望突出科技感、智能化和高端感。
```

系统输出：

- 抖音 AIGC/视觉创意达人；
- 懂车帝/汽车垂类测评达人；
- 小红书城市生活方式达人；
- 预热、背书、种草三阶段策略；
- 报价判断；
- 评论区价格争议风险；
- AIGC 内容真实性风险；
- 可推送合作 brief。

### 12.2 高端护肤新品种草

品牌输入：

```text
我们是高端护肤品牌，预算 30 万，准备推广一款抗老精华。
目标用户是 28-40 岁一二线城市女性，希望做真实种草和成分科技感表达。
```

系统输出：

- 小红书护肤种草达人；
- 抖音护肤测评达人；
- 成分党/精致女性人群匹配；
- 硬广风险提示；
- 价格和案例背书判断。

### 12.3 AI 软件品牌破圈

品牌输入：

```text
我们是一家 AI 软件公司，预算 20 万，希望在 B 端和 C 端之间做品牌破圈。
希望找科技、职场、商业、AIGC 内容达人，突出效率提升和职业变化。
```

系统输出：

- 科技类达人；
- 职场教育达人；
- AIGC 创意达人；
- 口播解释型内容建议；
- B 端信任和 C 端传播平衡建议。

## 14. 成功标准

第一版不追求完整交易闭环，也不承诺真实预测 ROI。

第一版成功标准：

1. 博主可以上传账号和合作案例；
2. 系统可以生成有用的商业标签；
3. 平台可以生成初步领域榜单；
4. 甲方可以输入 brief；
5. 系统可以推荐匹配博主；
6. 推荐结果有评分、理由、风险和预算建议；
7. 系统可以生成一份专业投放方案；
8. 用户能明确感知：这不是表格，而是一个 PR 决策系统。

## 17. 核心结论

PR AI OS 的产品故事不是“帮品牌找博主”。

更准确的故事是：

> **把 PR 行业里原本靠人脉、经验、表格和微信群完成的传播判断，升级成一个 AI 可理解、可匹配、可推演、可复盘的操作系统。**

它同时解决两侧问题：

- 对品牌：用更低成本获得更好的 PR 决策；
- 对博主：用案例和标签提升自己的商业价值；
- 对平台：沉淀跨行业、跨平台、跨案例的 PR 决策资产。

最终产品形态是：

```text
博主商业资产库
      +
品牌传播 Brief 输入器
      +
AI 匹配评分与榜单引擎
      +
投放策略和风险推演系统
      +
合作需求分发与案例回流
```

这就是 PR 行业的 AI OS 层。
