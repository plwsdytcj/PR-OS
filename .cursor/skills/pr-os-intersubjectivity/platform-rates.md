# PR-OS 平台刊例结构

新增达人与达人详情页必须使用同一套 `PLATFORM_RATE_FIELDS`（`web/static/app.js`）。

| 平台 | 报价项 |
|------|--------|
| 知乎 | 代发报价、原创发报价 |
| 抖音 | 60秒以内报价、120秒以内报价、长视频报价 |
| 微博 | 转发报价、原创发布报价 |
| 微信公众号 | 头条报价、次条报价、其他报价 |
| 豆瓣 | 短评打分报价、长评报价 |
| 小红书 | 笔记报价、视频报价 |
| 今日头条 | 微头条报价、图文报价 |
| 视频号 | 60秒以内报价、120秒以内报价 |
| 推特 | quote转发报价、thread讨论报价、原创发布报价、推特文章报价 |
| 其他 | 基础报价 |

## 存储约定

- 主报价写入 `listed_price`（取 primary 项或首个非零项）
- 分项写入 `manual_notes`：`{标签}：{金额}`，如 `代发报价：3000`
- 合作形式标签从报价项名称推导（去掉「报价」后缀）并合并进 `cooperation_formats`

## 平台列表

`PLATFORM_OPTIONS`：抖音、小红书、B站、微博、视频号、微信公众号、知乎、豆瓣、今日头条、推特

新增平台时须同时更新：`app.js`、`brief_parser.py`、`mapper.py`、`link_connector.py`、`server.py` 文本解析候选。
